REVISED ENTERPRISE SOLAR PROGRAMME INTERFACE AND WORKFLOW PLAN
1. Module Purpose

The Enterprise Solar Programme Module must enable a programme developer, sponsor, programme administrator and beneficiary organisations to create, plan and implement large-scale solar programmes across multiple beneficiaries.

The programme may cover:

Residential homes
Residential estates
Secondary schools
Hospitals and health facilities
Ministries
Regional offices
District assemblies
Farms
Irrigation facilities
Cold-storage facilities
Universities
Factories
Communities
Mini-grids
Utility-scale generation stations

The module must be part of the existing SolarPro application. It must not be built as a separate application.

The interface must be simple, guided and largely automated. Users should complete activities through:

Dropdown lists
Searchable selection lists
Checkboxes
Radio buttons
Toggle controls
Date pickers
Map selection
File uploads
Electricity-bill uploads
Site photographs
Voice notes
Review buttons
Approval buttons

Long text entry must be avoided wherever possible.

AI agents must automatically analyse information, create tasks, perform calculations and generate programme reports and deliverables.

2. Main Enterprise Entry Workflow

The user signs into SolarPro and selects Enterprise from the main application navigation.

The system checks whether the user belongs to an approved enterprise organisation.

Case 1: Organisation Not Registered

The user is redirected to:

Register Your Organisation

The organisation must be registered and approved before the user can create a programme.

Case 2: Organisation Already Registered

The user is redirected to:

Enterprise Programme Home

The Enterprise Programme Home displays the history of all programmes created by the user’s organisation.

Main Flow

SolarPro Login
→ Click Enterprise
→ Organisation validation
→ Organisation registration where required
→ Organisation approval
→ Enterprise Programme Home
→ Programme history
→ Create Programme or Open Existing Programme

21. Registration Hierarchy

The system must enforce this hierarchy:

Platform Administrator
→ Programme Developer Organisation
→ Programme
→ Programme Administrator
→ Beneficiary Organisation
→ Beneficiary Organisation Administrator
→ Beneficiary Organisation User

Registration Rules
The programme developer registers an organisation.
The programme developer organisation is approved.
The approved organisation creates a programme.
A Programme Administrator is assigned.
The Programme Administrator registers beneficiary organisations.
A Beneficiary Organisation Administrator is nominated or invited.
The Beneficiary Administrator completes the organisation profile.
The Programme Administrator approves the beneficiary organisation.
The Beneficiary Administrator invites beneficiary organisation users.
Users can only access the programmes, sites and tasks assigned to their organisation and role.

45. Core Acceptance Criteria

The revised module is acceptable only when:

A programme developer can register an organisation.
The approved programme developer can open Enterprise.
Programme history is displayed.
A programme can be created through dropdowns and guided selections.
The lifecycle phases appear at the top of the Programme Workspace.
Each phase opens a phase page.
Each phase displays deliverable buttons.
Clicking a deliverable generates agent and human actions.
Agents generate draft programme documents.
Users can review, approve or return documents.
Sponsor approval can unlock the next phase.
A Programme Administrator can register beneficiary organisations.
A Beneficiary Administrator can complete an invited registration.
A Beneficiary Administrator can invite organisation users.
A Beneficiary User can upload at least three bills.
The system supports six or more bill uploads.
Bill values are automatically extracted.
Bill values can be confirmed or corrected.
The Programme Administrator can run Check My Bill.
Monthly and average consumption are calculated.
Preliminary PV, battery and inverter capacity are estimated.
A Bill Analysis Report is generated.
The system recommends Standard Design or Generation Station routing.
A Standard Design output can be generated.
A Generation Station output can be generated.
Planning reports are generated automatically.
Outputs appear on the correct role-based taskboards.
The programme remains tenant-scoped.
Beneficiary organisations cannot view each other’s information.
All approvals and changes are auditable.
The workflow is tested from programme creation through planning approval.
The feature is integrated into the existing SolarPro application.
