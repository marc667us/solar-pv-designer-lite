# AI 3D Solar Shading Simulation Module

## Senior AI App Engineering Plan and Claude Code Implementation Markdown

## 1. Objective

Build a professional, data-driven, interactive 3D solar shading simulation module that can visually reproduce the reference dashboards for:

1. Residential roof-mounted PV with building obstruction
2. Residential roof-mounted PV with tree obstruction
3. Residential roof-mounted PV with water tank obstruction
4. Ground-mounted solar farm with building obstruction
5. Multiple obstructions combined shading
6. Hill obstruction
7. Cluster of buildings obstruction

The module must not generate random decorative 3D graphics. It must use real solar project data, site data, obstruction dimensions, PV layout, sun position, sun trajectory, and shadow-calculation outputs.

---

# 2. Engineering Approach

The correct architecture is:

```text
Solar Project Data
      ↓
Scene Template Selector
      ↓
3D Scene Data Normalizer
      ↓
Sun Position Engine
      ↓
Obstruction Geometry Builder
      ↓
PV Array Geometry Builder
      ↓
Shadow Projection Engine
      ↓
Panel Impact Engine
      ↓
3D Renderer
      ↓
Dashboard UI
      ↓
Report Export
```

Claude Code must not attempt to create the whole scene directly from text. It must build a reusable 3D engine using scene templates.

# 7. Scene Template Selection Logic

Create `sceneTemplateSelector.ts`.

```ts
export function selectSceneTemplate(input: SolarShading3DInput): ShadingSceneType {
  const obstructionTypes = input.obstructions.map(o => o.type);

  if (input.site.terrainType === "hill" || obstructionTypes.includes("hill")) {
    return "hill_obstruction";
  }

  if (input.site.terrainType === "urban_cluster" || obstructionTypes.filter(t => t === "cluster_building" || t === "building").length >= 3) {
    return "cluster_of_buildings";
  }

  if (input.obstructions.length > 1) {
    return "multiple_obstructions";
  }

  if (input.pvArray.mountingType === "ground_mounted" && obstructionTypes.includes("building")) {
    return "ground_mounted_building";
  }

  if (input.pvArray.mountingType === "roof_mounted" && obstructionTypes.includes("tree")) {
    return "residential_roof_tree";
  }

  if (input.pvArray.mountingType === "roof_mounted" && obstructionTypes.includes("water_tank")) {
    return "residential_roof_water_tank";
  }

  return "residential_roof_building";
}
```

# 8. Coordinate System

Use a strict coordinate system.

```text
X axis = East / West
Y axis = Height
Z axis = North / South

PV array centre = [0, 0, 0]

East  = +X
West  = -X
North = -Z
South = +Z
```

Direction mapping:

```ts
export function directionToVector(direction: string): [number, number, number] {
  const d = direction.toUpperCase();

  const map: Record<string, [number, number, number]> = {
    N: [0, 0, -1],
    S: [0, 0, 1],
    E: [1, 0, 0],
    W: [-1, 0, 0],
    NE: [0.707, 0, -0.707],
    NW: [-0.707, 0, -0.707],
    SE: [0.707, 0, 0.707],
    SW: [-0.707, 0, 0.707],
    WNW: [-0.92, 0, -0.38],
    NNW: [-0.38, 0, -0.92],
    ENE: [0.92, 0, -0.38]
  };

  return map[d] ?? [0, 0, 1];
}
```

# 9. Sun and Shadow Geometry

Use the following formulas as the first reliable engineering approximation.

```ts
export function solarVector(altitudeDeg: number, azimuthDeg: number): [number, number, number] {
  const alt = THREE.MathUtils.degToRad(altitudeDeg);
  const azi = THREE.MathUtils.degToRad(azimuthDeg);

  const x = Math.sin(azi) * Math.cos(alt);
  const y = Math.sin(alt);
  const z = Math.cos(azi) * Math.cos(alt);

  return [x, y, z];
}

export function shadowLength(height: number, altitudeDeg: number): number {
  const alt = THREE.MathUtils.degToRad(Math.max(altitudeDeg, 1));
  return height / Math.tan(alt);
}
```

Shadow direction:

```text
Shadow direction = opposite direction of sun ground vector
```
