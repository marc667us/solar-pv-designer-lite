Backup & Restore

Database: {{ backend }}
Backups are encrypted with a passphrase you choose and downloaded straight to your device — nothing is stored on the server. The passphrase is never saved; if you lose it the backup cannot be restored. Keep the file somewhere safe (it contains all platform data). If a table cannot be read the whole backup fails rather than producing a silent partial. Restore validates every table/column against the live schema and runs in one transaction that rolls back completely on any error.
