# Route-auth allowlist -- routes intentionally reachable without an auth
# DECORATOR. Consumed by tests/test_route_auth_ratchet.py.
#
# THE POINT OF THIS FILE
#   The ratchet asserts: every route with no auth decorator appears here.
#   Adding a new unprotected route FAILS CI until someone writes down why it
#   is public. It is a speed bump against accidental exposure, not a claim
#   that everything below is safe.
#
# FORMAT
#   <METHODS> <path>  # reason=<tag> [ref=<file:line>]
#
#   Methods are comma-separated, no spaces, and are PART OF THE KEY on
#   purpose: adding a POST to a GET-only public route trips the ratchet
#   instead of inheriting its exemption.
#
#   reason= is a CLOSED vocabulary, enforced by the tests:
#     public-by-design  anonymous access is the intended product behaviour
#     in-body-token     guarded by a bearer/shared-secret check INSIDE the
#                       handler, which a decorator scan cannot see.
#                       REQUIRES ref= naming where the check lives.
#     webhook-hmac      guarded by provider signature verification.
#                       REQUIRES ref=.
#     known-gap         a real gap, accepted for now and scheduled. These
#                       are the ONLY entries the sensitive-prefix test
#                       tolerates -- see test_sensitive_prefixes_are_protected.
#
#   ref= files are asserted to exist, so a rename cannot leave a dangling
#   justification behind.
#
# Baseline 2026-07-20 against HEAD d1b1965. Duplicate route declarations
# exist in source because new_*.py modules are byte-spliced into web_app.py;
# they collapse to one key here and one rule at runtime.
